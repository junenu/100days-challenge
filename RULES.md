# 100 Days Challenge — ルール

## 概要

100 日間、毎日 1 つのプロダクトを完成させる挑戦です。
言語・ジャンル不問。ただし **動くもの** を毎日届ける。

---

## 基本ルール

### 1. 毎日 1 プロダクト

- 1 日 1 ディレクトリ、1 プロダクト
- その日中に **完成の定義（DoD）** をすべて満たすこと（後述）
- 未完成のまま翌日に持ち越さない

### 2. ユニーク性

- 過去の日と同じプロダクトは作らない
- **別作と見なす基準**：以下の 4 項目のうち **3 項目以上** が異なること
  1. テーマ（例: タスク管理 / 天気 / ゲーム）
  2. 解決する課題（例: 記録 / 可視化 / 自動化）
  3. 主要機能（例: CRUD / リアルタイム更新 / 音声入力）
  4. 主要技術（例: Go / React / PyTorch）
- 類似ジャンルを続ける場合は上記基準を必ず確認すること

### 3. スタンドアローン動作

- 各プロダクトはそのディレクトリ単体で動作すること
- 別の day のコードに依存しない
- 外部 API・DB・モバイルランタイムへの依存は可。ただし **モックまたはサンプルデータでローカル最低限動作すること**
- README に起動手順を必ず記載する

---

## 完成の定義（DoD）

以下の **5 項目すべて** を満たした時点を「完成」とする。

| # | 条件 | 確認方法 |
| --- | ------ | --------- |
| 1 | README が存在し、概要・起動手順が記載されている | ファイル確認 |
| 2 | セットアップ〜起動までのコマンドがローカルで通る | 手元で実行 |
| 3 | 主要ユースケース 1 件以上が正常動作する | 手動確認 |
| 4 | エラー時の動作が確認できる（クラッシュしない） | 異常入力テスト |
| 5 | シークレットがコードに含まれていない | grep / git diff 確認 |

## ディレクトリ構造

```
100days-challenge/
├── RULES.md           # 本ファイル
├── README.md          # 全体の進捗一覧
├── day001_<name>/
│   ├── README.md      # 概要・起動手順・開発ログ
│   └── ...            # プロダクトのソースコード
├── day002_<name>/
│   └── ...
└── ...
```

### 命名規則

```
day{NNN}_{product-name}
```

- `NNN`: ゼロ埋め 3 桁（001〜100）
- `product-name`: ケバブケース、英小文字
- 例: `day001_todo-cli`, `day042_weather-dashboard`

## 品質基準

### 必須（DoD 達成に必要）

- [ ] ローカルで起動できる
- [ ] コアの機能が動作する
- [ ] README に起動手順が書かれている
- [ ] シークレット（API キー・パスワード）をハードコードしない → `.env` または環境変数を使用
- [ ] エラーハンドリングを必ず実装

### 条件付き必須

- [ ] 外部設定（API キー等）がある場合のみ：`.env` ファイルを `.gitignore` に追加し、`.env.example` を提供
- [ ] SQL を使う場合：パラメータ化クエリを使用
- [ ] Web アプリの場合：XSS / CSRF 対策を実施
- [ ] ユーザー入力がある場合：バリデーションを実装

### 推奨（品質向上）

- [ ] 関数は 50 行以内、ファイルは 800 行以内
- [ ] マジックナンバーは定数化
- [ ] イミュータブルなデータ操作を優先

## 進捗管理

プロダクト完成時（日次完了コミットと同タイミング）に、ルート `README.md` の進捗テーブルへ 1 行追加する。
この追加は **DoD の達成条件** に含まれる。

```markdown
| Day | プロダクト名 | 技術スタック | 一言メモ |
|-----|------------|-------------|---------|
| 001 | todo-cli   | Go          | シンプルな TODO 管理 CLI |
```

---

## コミット規約

```
feat(dayNNN): <プロダクト名> — <一行説明>
```

例:

```
feat(day001): todo-cli — Go製のシンプルTODO管理CLIを追加
feat(day042): weather-dashboard — OpenWeatherMap連携のリアルタイム天気表示
```

---

## 禁止事項

以下は **日次完了コミット** に対して適用する。

- 過去の day の丸コピー・微修正はカウントしない
- DoD（完成の定義）を満たしていない状態でのコミットは禁止
- プロダクトの説明のない README だけのコミットは禁止
- シークレットを含むコミットは禁止

---

## ゴール

100 日後に、100 種類の動くプロダクトが揃っていること。
それぞれが独立して価値を持ち、誰でも手元で動かせること。
